Résumé des instructions

  • Application de blog simple en PHP avec une base de données MySQL
  • Interface Frontend (lecture de billets)
  • Interface Backend (administration des billets pour l'écriture) protégée par un mot de passe
  • CRUD : création lecture, mise à jour et suppression de billets
  • Chaque billet doir permettre l'ajout de commentiares qui peuvent être modérés via l'interface d'administration
  • Les lecteurs doivent pouvoir "signaler" les commentiares pour que ceux-ci remontent plus facilement dans l'interface d'administration pour être modérés
  • La rédaction de billets se fera dans une interface WYSIWYG basée sur TinyMCE
  • PHP sans framework, architecture MVC
  • Développer autant que possible en orienté objet (au minimum le modèle doit être construit sous forme d'objet
